# 角色立绘悬浮窗（st-stage）

给**任何纯文字角色卡**加上视觉小说式的立绘悬浮窗：插件悄悄告诉 AI「回复末尾带上表情标签」，然后从 AI 回复里读出 `[立绘:微笑]` 这样的标签，实时切换悬浮窗里的角色立绘。不需要角色卡作者做任何配合，也不依赖表情识别模型。

支持：一条消息**多张立绘序列播放**（点击/自动轮播）、**一个聊天启用多个图包 + 三级地址寻址**（`[立绘:鸣人/居家服/微笑]`）、**每次回复输出 N 张立绘**、**imgbb 图床自动直传**、图库管理与两种分享方式（JSON / 一行分享串 stpack2）、**可逆楼层渲染**（关掉立绘旧楼层立即恢复原文）。

同时内置一个「小手机」作为统一功能入口：聊天界面有个可拖动的 📱 图标，点开是一部小手机，立绘设置、图库都是里面的 App——以后其他工具也直接开发进这个手机。不喜欢手机形态也可以在扩展设置里关掉手机，但立绘功能不受影响。

## 安装

1. 打开 SillyTavern，点顶部的 **扩展**（Extensions，积木图标）
2. 点 **安装扩展**（Install extension）
3. 粘贴本仓库地址，点安装：

```
https://github.com/LYC619/st-stage
```

4. 安装完成后，聊天界面会出现两个新东西：
   - 一个**立绘悬浮窗**（初次是「未绑定立绘包」的占位提示）
   - 一个可拖动的 **📱 悬浮图标**（小手机入口，立绘设置和图库都在里面）

> ⚠️ **更新扩展后界面没变化？**多半是浏览器缓存了旧文件：电脑 Ctrl+F5 强制刷新；手机浏览器请在站点设置里清除缓存后重进（ST 加载扩展文件没有缓存刷新机制）。

> 📍 **设置在哪？** ST 扩展设置页只留两个总开关（**启用立绘功能**、**显示手机**）。立绘的显示方式、楼层数、轮播、Prompt 模式在手机「🎭 立绘」App 里；图包管理、上传、分享、imgbb/图床设置在手机「🗂 图库」App 里。

## 三分钟上手（用内置立绘包）

插件自带两套立绘包：「银发萝莉」和「黑长直御姐」，各 8 个表情，装完就能用。

1. **打开任意一个角色的聊天**
2. 点聊天界面的 **📱 图标** → **🗂 图库** → **打开图库管理**
3. 最上面「当前角色」区，下拉选 **银发萝莉（8 张）**（或御姐）启用；一个聊天可以启用多个包
4. 关掉弹窗，**发一条消息**

AI 回复时，插件会自动在提示词里附上「可用表情列表」，AI 会在回复末尾写 `[立绘:害羞]` 之类的标签，悬浮窗立刻切换到对应表情。就这么简单。

> 💡 悬浮窗可以**拖动**（按住拖）和**缩放**（右下角拖拽），位置大小会自动记住。手机小屏也适配：位置会按真实尺寸完整收回屏幕内，触屏可拖拽/缩放。悬浮窗右上角有 **✕ 关闭**按钮，只隐藏窗体（立绘功能不受影响，可到手机「立绘」App 重新打开）。
> 💡 如果不想在聊天文字里看到 `[立绘:xxx]` 标签：手机 → 🎭 立绘 App → 勾选「隐藏 [立绘:xxx] 标签」。
> 💡 不喜欢悬浮窗？「立绘」App 里把「**显示位置**」切成「**仅楼层**」，立绘会直接显示在聊天气泡里 `[立绘:xxx]` 标签的位置（本地上传、内嵌、图床图源都支持）；也可以选「两者」。**仅楼层模式下绝不弹悬浮窗。**
> 💡 一条回复里有**多个** `[立绘:xxx]` 标签时，悬浮窗会按顺序排成一个序列：底部出现圆点指示，**点击图片切下一张**；「立绘」App 里可开「自动轮播」（间隔秒数可调）。想让 AI 每次输出多张？把「**每次回复立绘数量**」调到 1–10。
> 💡 **可逆楼层渲染**：切换显示模式、关掉总开关、关掉隐藏标签时，已经改过的旧楼层会立即恢复成原始文字，不用等 ST 重新渲染。默认只补渲染最近 **6** 个 AI 楼层（可在「立绘」App 调 1–50）。

## 用自己的图做立绘包

准备几张表情图（PNG/JPG/WebP 都行），**把文件名改成表情名**，比如：

```
微笑.png   害羞.png   生气.png   哭泣.png   惊讶.png
```

> 表情名会直接进 AI 的提示词，所以起名越直白越好（微笑、害羞、生气…），AI 才容易选对。

然后：

1. 手机 → 🗂 图库 → **打开图库管理**
2. 底部输入框输入包名（比如「我的老婆」）→ 点 **新建立绘包**，自动进入包详情页
3. 点 **批量上传（自动压缩+解析预览）**，一次全选你的图 → 会先弹**解析预览**，确认无误再上传。文件名自动成为表情名，图片自动压缩成 WebP（最长边 1024px），存到 ST 的用户数据目录（**以后更新插件不会丢图**）
4. 返回列表，把当前角色绑定到这个包

详情页里还能对每张图：**✎ 改名、🏷 设分组、🖼 替换、★ 设封面、◀▶ 排序、✕ 删除**（鼠标悬停在缩略图上出现按钮）。

## 一个聊天启用多个图包 + 三级地址（多角色 / 换装）

一张卡里有多个角色（火影同人卡的鸣人、佐助），或者想给同一个角色做不同衣服的立绘？现在**一个聊天可以同时启用多个图包**，用**三级地址**精确寻址：

- `[立绘:微笑]` —— 图名（只启用一个包时的简写）
- `[立绘:鸣人/微笑]` —— 人名/图名
- `[立绘:鸣人/居家服/微笑]` —— 人名/服装/图名

**两种组织方式：**

1. **一包一角色/服装**（推荐）：给包设「人名」「服装」（包信息里填），包内立绘用纯图名。批量上传时文件名 `鸣人-居家服-微笑.png` 会自动建/配到「鸣人·居家服」包。
2. **一包多分组**（旧方式，继续兼容）：一个包里用「分组」区分多个人名，`鸣人_微笑.png` 拆成分组「鸣人」。旧包详情页有「**按分组拆成立绘包**」一键升级成方式 1（先预览、原包保留）。

**绑定多个包**：图库管理顶部「当前角色」区，下拉可以逐个**启用多个包**，支持**上移/下移排序**和单独**停用**。

**严格寻址**：`[立绘:鸣人/微笑]` 只会在鸣人名下找「微笑」；找不到就不显示，**绝不会串到佐助的微笑**。

**每次回复立绘数量**（「立绘」App 里设，1–10）：设成 N，Prompt 会要求 AI「按情节顺序选择 N 张立绘，依次输出 N 个标签」，悬浮窗/楼层按顺序展示。

**两种 Prompt 模式**（「立绘」App 里选）：

| 模式 | 适合 | 说明 |
| --- | --- | --- |
| 全量 | 各角色表情名不统一 | 把所有完整地址列进提示词 |
| 智能精简 | 各角色用同一套表情名 | 共有图名合并列出，其余放「其他图片」写完整地址（不生成不存在的组合） |

> 💡 单角色换装同样适用：服装填「校服」「泳装」，AI 就能用 `[立绘:泳装/害羞]` 切换装扮。

## 分享立绘包给别人

有两种方式，按需选：

### 方式一：JSON 文件（最简单，适合本地图）

包详情页 → **导出 JSON**，得到一个 `包名.sprite-pack.json` 文件，发给对方。本地上传的图片会自动打包进文件里（base64 内嵌），对方拿到文件后：管理弹窗 → **导入 JSON 文件** → 选中即用，图都在。

缺点：图多的时候文件可能好几 MB。

### 方式二：一行分享串（推荐，适合发聊天群）

分享串（stpack2）长这样，一行文本，每张图带完整独立 URL、支持三级地址：

```
stpack2:我的老婆|微笑=https://i.ibb.co/aa/smile.png|鸣人/居家服/害羞=https://i.ibb.co/bb/x.png
```

对方复制这行字 → 图库管理底部「粘贴 stpack2:/stpack1: 分享串」→ 点 **导入分享串**，整个包就到手了（图片从图床加载，不占对方存储）。三级地址（人名/服装）会一并还原。

**制作步骤（最省事：imgbb 自动直传）：**

1. 到 [api.imgbb.com](https://api.imgbb.com/) 免费申请 API Key，填进「🗂 图库」App，勾选「上传立绘时自动直传 imgbb」
2. 正常批量上传图片——本地保存的同时自动传 imgbb 拿到远程地址
3. 详情页点 **复制分享串** → 发到群里 ✅

> ✅ **完整性预检**：复制分享串前会检查每张图是否都有远程地址。如果有图还没上传（只有本地），会提示 `N/M 张有远程地址` 和缺失清单，需要你**确认**才复制——**绝不会静默生成残缺分享串**。详情页有「图床补传」按钮可把待上传的图一键补传 imgbb（失败可重试）。
> 💡 也可以手动传 [catbox.moe](https://catbox.moe) 等图床，用详情页「按编码添加」逐张录入。
> 💡 本地图/内嵌图的包，分享用「导出 JSON」（图 base64 内嵌在文件里）；继续兼容导入旧的 stpack1 分享串。

## 消息内插图（进阶玩法，可选）

除了角色立绘，还可以让 AI 在回复里**插入场景/CG 图**。开启后，AI 回复中的这两种写法都会被渲染成图片：

```
[插图:ab12cd.png]        ← 推荐写法
<img>ab12cd.png</img>    ← 兼容社区正则玩法的写法
```

开启方式：扩展设置 → 勾选「渲染消息内插图」。编码同样是图床文件名，拼上你设置的图床前缀显示。

怎么让 AI 主动输出这种标记？在角色卡的系统提示词或世界书里告诉它可用的图和编码即可，例如：

```
你可以在合适的场景插入以下图片，格式为 [插图:编码]：
教室场景 = jk98xy.png
放学的夕阳 = mn45ab.png
```

## 手机 📱 是干什么的

点聊天界面的 📱 图标展开小手机，它是 st-stage 各功能的**统一入口**。目前有两个 App：

| App | 功能 |
| --- | --- |
| 🎭 立绘 | 立绘设置中心：总开关、显示位置（悬浮窗/仅楼层/两者）、悬浮窗显隐、最近渲染楼层数、每次回复立绘数量、多立绘轮播与间隔、隐藏立绘标签、消息内插图、Prompt 模式 |
| 🗂 图库 | 图包创建/编辑/删除、当前聊天启用多个图包、上传与批量解析、旧分组拆包、导入/导出/分享串、imgbb 与图床设置 |

手机顶部左上角 **‹ 返回**、右上角 **✕ 关闭**（只收起手机，恢复成可拖动图标），底部圆形 **Home 键**返回主屏。手机图标可拖到任意位置，按真实尺寸完整收进屏幕。

> 💡 不喜欢小手机？ST 扩展设置 → 取消勾选「**显示手机**」。关掉手机不影响立绘功能，图库仍可从悬浮窗 ⚙ 打开。
> 💡 从悬浮窗 ⚙ 打开图库弹窗时，关闭后正常关掉；从手机「图库」打开时，关闭后会自动回到手机图库页。

**开发者**：以后其他工具直接开发成手机内的 App 模块，与 st-stage 一起构建发布（内部装配见 `st-extension/src/apps/`）。历史的 `window.stStage.registerApp(...)` 外部注册入口暂时保留兼容，详见 [docs/APP-SPEC.md](docs/APP-SPEC.md)。

## 常见问题

**Q：AI 不输出 [立绘:xxx] 标签？**
先确认角色已绑定立绘包且绑定是「启用」状态（没绑定就不会注入提示词）。个别模型偶尔忘写标签属正常现象，下一条通常就恢复；也可以在角色卡里补一句「记得每次回复末尾标注表情」加强。

**Q：换了角色，立绘没变？**
插件按**角色名**绑定立绘包，每个角色要各自绑定一次。切角色后悬浮窗会自动切到该角色启用的包（没绑定则显示占位提示）。旧的单包绑定会自动迁移成新的多包绑定，不丢数据。

**Q：更新插件后我上传的图会丢吗？**
不会。图片存在 ST 的用户数据目录（`/user/images/sprite-overlay/…`），不在插件目录里。设置也会自动迁移（旧单包绑定 → 多包、旧分组包、stpack1 都兼容）。

**Q：分享串导入后图片显示不出来？**
分享串依赖图床在线加载——检查网络能否访问 catbox（部分地区需要代理）；或让分享者确认图床文件还在。

**Q：悬浮窗/手机拖不见了？**
手机 → 立绘 App → 「把悬浮窗拉回视口」。手机图标本身出界会被自动收回屏幕内。

**Q：关掉立绘功能后，之前楼层里替换过的立绘怎么办？**
关掉总开关（或切回悬浮窗模式、关掉隐藏标签）时，改过的旧楼层会**立即恢复成原始文字**，不用刷新。手机本身不受总开关影响，照常可用。

## 本地开发

网页版是仿 ST 的聊天模拟器，用于本地开发和测试核心链路（注入 → 模拟 AI 回复 → 提取标签 → 切换立绘）：

```bash
pnpm install
pnpm dev        # 启动网页测试环境 http://localhost:3000
pnpm test       # core 层单元测试（vitest）
pnpm lint       # ESLint
pnpm typecheck  # tsc --noEmit
pnpm build:ext  # 重新打包 ST 扩展（产物输出到根目录 index.js / style.css，需提交）
```

### 目录结构

| 路径 | 说明 |
| --- | --- |
| `manifest.json` / `index.js` / `style.css` | ST 扩展产物（根目录，供 GitHub 链接安装） |
| `core/` | 平台无关核心逻辑（标签解析、prompt 构建、包管理、导入导出、分享串、迁移、压缩、手机框架）+ 单元测试 |
| `st-extension/src/` | ST 扩展源码（适配器 + 原生 DOM UI），经 esbuild 打包为根目录 `index.js` |
| `app/` `components/` `lib/` | Next.js 网页测试环境 |
| `public/presets/` | 内置预设立绘图片（随扩展安装一起分发） |
| `docs/APP-SPEC.md` | 手机 App 开发规范（内部 App 模块为主，兼容旧的第三方注册） |

> 注意：修改 `core/` 或 `st-extension/src/` 后必须运行 `pnpm build:ext` 并提交根目录产物，GitHub 安装的用户才能拿到更新。

### 数据格式

- 设置存储：`extensionSettings.sprite_overlay`，`settingsVersion: 3`，旧版本自动迁移（新字段缺失自动补默认值；旧单包绑定 `packId` → 多包 `packIds`；幂等）
- 立绘：`{ tag, url, code?, group?（人名）, outfit?（服装）, remoteUrl?（远程分享地址） }`——本机显示优先 `url`（本地保底），分享用 `remoteUrl`
- 立绘包：`{ id, name, author?, description?, roleName?（人名）, outfit?（服装）, coverTag?, sprites[] }`
- 三级地址：`图名` / `人名/图名` / `人名/服装/图名`；人名取 `sprite.group || pack.roleName`，服装取 `sprite.outfit || pack.outfit`
- 立绘包文件：`sprite-pack@2`（含 roleName/outfit/group/outfit 字段），导入兼容 `sprite-pack@1`
- 分享串：`stpack2:包名|@author=…|地址=完整URL|…`（每图独立 URL、三级地址）；继续兼容导入 `stpack1`
