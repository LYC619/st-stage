# 角色立绘悬浮窗（st-stage）

给**任何纯文字角色卡**加上视觉小说式的立绘悬浮窗：插件悄悄告诉 AI「回复末尾带上表情标签」，然后从 AI 回复里读出 `[立绘:微笑]` 这样的标签，实时切换悬浮窗里的角色立绘。不需要角色卡作者做任何配合，也不依赖表情识别模型。

同时内置一个「手机」框架：聊天界面有个可拖动的 📱 图标，点开是一部小手机，立绘、图库、设置都是里面的 App——第三方扩展也可以把自己的功能注册成 App（见 [docs/APP-SPEC.md](docs/APP-SPEC.md)）。

---

## 安装

1. 打开 SillyTavern，点顶部的 **扩展**（Extensions，积木图标）
2. 点 **安装扩展**（Install extension）
3. 粘贴本仓库地址，点安装：

```
https://github.com/LYC619/st-stage
```

4. 安装完成后，聊天界面会出现两个新东西：
   - 一个**立绘悬浮窗**（初次是「未绑定立绘包」的占位提示）
   - 一个可拖动的 **📱 悬浮图标**（手机入口）

---

## 三分钟上手（用内置立绘包）

插件自带两套立绘包：「银发萝莉」和「黑长直御姐」，各 8 个表情，装完就能用。

1. **打开任意一个角色的聊天**
2. 鼠标移到立绘悬浮窗上，点右上角的 **⚙**，打开「立绘包管理」
3. 最上面一行「角色 XXX 绑定」，下拉框里选 **银发萝莉（8 张）**（或御姐）
4. 关掉弹窗，**发一条消息**

AI 回复时，插件会自动在提示词里附上「可用表情列表」，AI 会在回复末尾写 `[立绘:害羞]` 之类的标签，悬浮窗立刻切换到对应表情。就这么简单。

> 💡 悬浮窗可以**拖动**（按住拖）和**缩放**（右下角拖拽），位置大小会自动记住。
> 💡 如果不想在聊天文字里看到 `[立绘:xxx]` 标签：扩展设置 → 角色立绘悬浮窗 → 勾选「消息中隐藏 [立绘:xxx] 标签」。

## 用自己的图做立绘包

准备几张表情图（PNG/JPG/WebP 都行），**把文件名改成表情名**，比如：

```
微笑.png   害羞.png   生气.png   哭泣.png   惊讶.png
```

> 表情名会直接进 AI 的提示词，所以起名越直白越好（微笑、害羞、生气…），AI 才容易选对。

然后：

1. 悬浮窗 ⚙ 打开管理弹窗
2. 底部输入框输入包名（比如「我的老婆」）→ 点 **新建立绘包**，自动进入包详情页
3. 点 **上传图片（自动压缩）**，一次全选你的图 → 完成。文件名自动成为表情名，图片自动压缩成 WebP（最长边 1024px），存到 ST 的用户数据目录（**以后更新插件不会丢图**）
4. 返回列表，把当前角色的绑定切到这个包

详情页里还能对每张图：**✎ 改名、🖼 替换、★ 设封面、◀▶ 排序、✕ 删除**（鼠标悬停在缩略图上出现按钮）。

## 分享立绘包给别人

有两种方式，按需选：

### 方式一：JSON 文件（最简单，适合本地图）

包详情页 → **导出 JSON**，得到一个 `包名.sprite-pack.json` 文件，发给对方。本地上传的图片会自动打包进文件里（base64 内嵌），对方拿到文件后：管理弹窗 → **导入 JSON 文件** → 选中即用，图都在。

缺点：图多的时候文件可能好几 MB。

### 方式二：一行分享串（推荐，适合发聊天群）

分享串长这样，就一行文本：

```
stpack1:我的老婆|微笑=ab12cd.png|害羞=xy34zw.png|生气=qw56er.png
```

对方复制这行字 → 管理弹窗底部「粘贴 stpack1: 开头的分享串」→ 点 **导入分享串**，整个包就到手了（图片从图床加载，不占对方存储）。

**制作步骤：**

1. 打开图床网站 [catbox.moe](https://catbox.moe)（免费、不用注册），把你的表情图一张张上传
2. 每张传完会得到一个链接，比如 `https://files.catbox.moe/ab12cd.png` —— 记下最后那段文件名 `ab12cd.png`（这就是「编码」），并记住它对应哪个表情
3. 回到插件：新建一个立绘包 → 详情页底部「按编码添加」：表情填 `微笑`，编码填 `ab12cd.png`，点添加。逐张加完
4. 点 **复制分享串** → 发到群里 ✅

> 💡 也可以先用本地图自己玩，确定要分享了再传图床、按编码重建一个「分享版」的包。
> 💡 默认图床是 catbox；如果你用别的图床，在扩展设置里改「图床前缀」，分享串会自动带上你的图床地址。

## 消息内插图（进阶玩法，可选）

除了角色立绘，还可以让 AI 在回复里**插入场景/CG 图**。开启后，AI 回复中的这两种写法都会被渲染成图片：

```
[插图:ab12cd.png]        ← 推荐写法
<img>ab12cd.png</img>    ← 兼容社区正则玩法的写法
```

开启方式：扩展设置 → 勾选「渲染消息内插图」。编码同样是图床文件名，拼上你设置的图床前缀显示。

怎么让 AI 主动输出这种标记？在角色卡的系统提示词或世界书里告诉它可用的图和编码即可，例如：

```
你可以在合适的场景插入以下图片，格式为 [插图:编码]：
教室场景 = jk98xy.png
放学的夕阳 = mn45ab.png
```

## 手机 📱 是干什么的

点聊天界面的 📱 图标展开小手机，里面目前有三个 App：

| App | 功能 |
| --- | --- |
| 🎭 立绘 | 看当前角色绑定了哪个包；点缩略图直接预览表情；一键把悬浮窗拉回屏幕（拖丢了的时候用） |
| 🗂 图库 | 打开立绘包管理弹窗（和悬浮窗 ⚙ 是同一个） |
| ⚙️ 设置 | 总开关、隐藏标签、插图渲染、图床前缀 |

手机图标同样可以拖到任意位置，按 Home 键（底部横条）返回主屏/收起手机。

**开发者**：手机是一个开放框架，你的 ST 扩展可以调用 `window.stStage.registerApp(...)` 注册自己的 App（掷骰器、状态面板、小游戏……），获得手机内的入口、页面容器和私有存储。接入只要十几行代码，完整规范见 [docs/APP-SPEC.md](docs/APP-SPEC.md)。

## 常见问题

**Q：AI 不输出 [立绘:xxx] 标签？**
先确认角色已绑定立绘包且绑定是「启用」状态（没绑定就不会注入提示词）。个别模型偶尔忘写标签属正常现象，下一条通常就恢复；也可以在角色卡里补一句「记得每次回复末尾标注表情」加强。

**Q：换了角色，立绘没变？**
插件按**角色名**绑定立绘包，每个角色要各自绑定一次。切角色后悬浮窗会自动切到该角色绑定的包（没绑定则显示占位提示）。

**Q：更新插件后我上传的图会丢吗？**
不会。图片存在 ST 的用户数据目录（`/user/images/sprite-overlay/…`），不在插件目录里。设置也会自动迁移。

**Q：分享串导入后图片显示不出来？**
分享串依赖图床在线加载——检查网络能否访问 catbox（部分地区需要代理）；或让分享者确认图床文件还在。

**Q：悬浮窗/手机拖不见了？**
手机 → 立绘 App → 「把立绘窗拉回视口」。手机图标本身出界会被自动收回屏幕内。

## 本地开发

网页版是仿 ST 的聊天模拟器，用于本地开发和测试核心链路（注入 → 模拟 AI 回复 → 提取标签 → 切换立绘）：

```bash
pnpm install
pnpm dev        # 启动网页测试环境 http://localhost:3000
pnpm test       # core 层单元测试（vitest）
pnpm lint       # ESLint
pnpm typecheck  # tsc --noEmit
pnpm build:ext  # 重新打包 ST 扩展（产物输出到根目录 index.js / style.css，需提交）
```

### 目录结构

| 路径 | 说明 |
| --- | --- |
| `manifest.json` / `index.js` / `style.css` | ST 扩展产物（根目录，供 GitHub 链接安装） |
| `core/` | 平台无关核心逻辑（标签解析、prompt 构建、包管理、导入导出、分享串、迁移、压缩、手机框架）+ 单元测试 |
| `st-extension/src/` | ST 扩展源码（适配器 + 原生 DOM UI），经 esbuild 打包为根目录 `index.js` |
| `app/` `components/` `lib/` | Next.js 网页测试环境 |
| `public/presets/` | 内置预设立绘图片（随扩展安装一起分发） |
| `docs/APP-SPEC.md` | 手机 App 开发规范（面向第三方开发者） |

> 注意：修改 `core/` 或 `st-extension/src/` 后必须运行 `pnpm build:ext` 并提交根目录产物，GitHub 安装的用户才能拿到更新。

### 数据格式

- 设置存储：`extensionSettings.sprite_overlay`，`settingsVersion: 2`，旧版本自动迁移
- 立绘包文件：`sprite-pack@2`（`{ format, name, author?, description?, coverTag?, sprites: [{ tag, url? | data?, code? }] }`），导入兼容 `sprite-pack@1`
- 分享串：`stpack1:包名|@host=…|@author=…|表情=图床文件名|…`（默认图床 catbox 时省略 @host）
